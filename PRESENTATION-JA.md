## 1. 90秒デモ・スクリプト(司会用)

> **タイミング目安**：実演 60秒 + 一言コメント 30秒 = 計90秒。
> **準備**：ブラウザを開き `?` のサンプルブリーフが表示される状態にしておく。右上の言語切替で「日本語」を選択しておく。

### (00:00) 導入 — 「3行のブリーフから、4つの広告素材を3分で。」

「これは、マーケティング担当者がプロンプトを書かずに、日本語のディレクションだけで広告クリエイティブを完成させるための『ディレクターズ・コックピット』です。代理店に1週間お願いしていた作業を、その場で完結させます。」

### (00:10) 1クリック復元 — 「まずはサンプルで全体像を。」

`サンプルブリーフを試す` をクリック。

「いま、コピー・画像・スクリプト・音声の4ステップが、キャッシュから瞬時に復元されました。これは『すでに一度作ったもの』を再生しています。本番では各ステップ20〜30秒です。」

### (00:25) ディレクション翻訳の実演 — 「ここが、他のAIツールとの決定的な違いです。」

ステップ1(コピー)に戻り、Refine欄に**日本語で**入力：

> 「もっと攻めた感じで、企業っぽさを抜いて」

`調整` をクリック。

「いま、皆さんは『プロンプト』を書いていません。日本語で監督に指示を出しただけです。裏側で当社の**ディレクション翻訳エンジン**が、この一言を構造化された制作指示に変換し、生成AIに渡しています。これがコアIPです。」

### (00:45) 監督による批評 — 「AIに『ダメ出し』をさせる。」

ステップ2(画像)で `批評` をクリック。Claude Sonnetが画像を視覚分析した日本語のフィードバックを表示。

「これはClaude(Anthropic社のAI)が画像を実際に『見て』、シニアクリエイティブディレクターとして批評を返しています。気に入れば `この批評を反映` でそのまま次の生成へ。AIが品質管理側にも回るのが特徴です。」

### (01:10) 音声まで日本語で — 「広告全体が、ネイティブの日本語で。」

音声ステップで日本語ボイスを選択 → `承認`。

「ElevenLabsの日本人ボイスで音声化されました。LINE広告、Yahoo!広告、TikTokなどへそのまま使えます。」

### (01:25) クロージング

「3分前は3行のブリーフだけでした。いまは、コピー・画像・スクリプト・音声、そしてすべての判断履歴を残した『ディレクターズノート』まで揃っています。代理店に発注すると1案件20〜50万円、納期は5営業日。これを社内で、3分で。」

## 2. 1ページ・ピッチ(配布用)

### Director's Cockpit / ディレクターズ・コックピット

**「プロンプトを書かない、日本語で指示するだけの広告制作AIパイプライン」**

#### 解決する課題

日本企業の広告制作には3つのボトルネックがあります。

1. **生成AIの使いこなしが属人化** — ChatGPT・Midjourneyを「使える人」と「使えない人」の差が広がっている。プロンプトエンジニアリングを全社員に学ばせるのは現実的ではない。
2. **品質のばらつき** — 個人が単発でAIを使うと、ブランドトーンが揃わない。制作物の「審査」も人手頼み。
3. **国内対応の遅れ** — 海外発のAIツールは英語前提。日本語の微妙なトーン(です・ます調、攻めた言い回し、和文タイポグラフィ)に弱い。

#### 当社の解決策

「ディレクション翻訳」レイヤーを介して、**日本語のラフな指示を、生成AIが理解できる構造化命令に自動変換**するパイプラインです。ユーザーは「もっと和風に」「お祭り感を出して」と話すだけ。プロンプトは書きません。

4ステップ(コピー→画像→スクリプト→音声)を一気通貫で。各ステップでユーザーは2案から選ぶか、日本語で再指示するか、AIによる批評を求めるか、を選びます。

#### なぜ今か (Why now)

- **基盤AIの精度がライン突破** — GPT-4o、Claude Sonnet、Flux、ElevenLabsが揃って実用品質に到達したのは2024〜2025年。
- **広告費の対GDP比は日本が世界最高水準** — 2024年の国内広告費は約7.7兆円(電通発表)。うち中小事業者・自治体向けに「代理店を介さず内製したい」需要が顕在化。
- **生成AI規制の枠組みが固まりつつある** — 経産省ガイドライン整備、EU AI法、米国大統領令により、「キーをサーバーに渡さない」「生成物の出所が追える」プロダクトに優位性。本製品は両方を満たします。

#### 競合優位性 (Moat)

| 項目                          | 一般的な生成AIツール     | Director's Cockpit |
| ----------------------------- | ------------------------ | ------------------------------------ |
| プロンプト記述                | 必要(英語が望ましい)     | 不要(日本語のラフ指示でOK)           |
| ワークフロー                  | 単発生成                 | 4ステップの編集可能パイプライン      |
| ブランドトーンの一貫性        | 都度プロンプトで再指定   | 上流ステップが下流を自動制約         |
| 制作判断の監査証跡            | なし                     | ディレクターズノートに全自動記録     |
| データ主権                    | サーバーに鍵を預ける     | ブラウザのセッション内で完結         |
| 履歴の取り直し                | やり直し=都度課金        | キャッシュ復元でゼロコスト           |

**核心は「ディレクション翻訳」エンジン**。 これは公開モデルでは置き換えできない、当社の少数事例ベースで作り込んだプロンプト群です(各アセット種別ごとに専用)。

#### ビジネスモデル(検討中)

1. **SaaS従量課金** — 月額固定 + 生成回数。社内チーム向け。
2. **ホワイトラベルOEM** — 代理店、印刷会社、SaaS事業者にOEM提供。
3. **エンタープライズ・オンプレ** — 銀行・官公庁向けにブランド辞書搭載のクローズド版。

#### 現状とロードマップ

現状はプロダクト・デモ(本日ご覧いただいた状態)。**英語・日本語の2言語対応・国内主要AI API連携・ブラウザ完結型**まで完了。

次の3〜6ヶ月で：ブランド辞書、チームレビュー、LINE広告・Yahoo!広告フォーマット出力対応、APIエンドポイント公開。

## 4. 想定質問と回答集

#### Q1. データはどこに保存されますか?当社の顧客情報・ブランド情報が漏れる懸念は?

A. **生成APIキー、入力テキスト、生成結果のいずれも、当社のサーバーには一切送信されません**。ブラウザのセッションストレージ内で完結し、タブを閉じれば消去されます。通信先は、お客様自身が契約された OpenAI、Anthropic、fal.ai、ElevenLabs のAPIエンドポイントのみです。

エンタープライズ版ではさらに、Azure OpenAI のお客様テナント、AWS Bedrock(東京リージョン)経由の構成も提供予定です。

#### Q2. 生成された画像・コピー・音声の著作権・利用権は?

A. 各生成プロバイダの利用規約に従います。商用利用可能なプランで API キーを契約していれば、生成物はお客様に帰属します(OpenAI、Anthropic、fal.ai/Flux、ElevenLabsいずれも商用利用可)。

なお、画像生成モデルの学習データに関する係争(米国Stability AI訴訟等)は把握しており、リスク選好に応じて、利用モデルを切り替え可能な設計にしています。

#### Q3. 1広告あたりのコストは?

A. **OpenAI + Anthropic + fal.ai + ElevenLabs の合算APIコストで、1パイプライン完走あたり概ね 5〜15円**(2025年時点・USD換算)。試算内訳：

- コピー生成(Claude Sonnet × 2案):約 ¥3
- 画像生成(Flux Schnell × 2枚):約 ¥4
- スクリプト生成(GPT-4o-mini × 2案):約 ¥1
- 音声合成(ElevenLabs 30秒):約 ¥5
- 画像批評(Claude Sonnet Vision):約 ¥2

代理店外注の場合、1案件20〜50万円。**実コスト比 1万倍以上の差**。社内人件費を加味しても、月100案件で数百万円の削減効果。

#### Q4. 既存のAdobe Firefly、Canva Magic Studio、ChatGPT、ELYZA、Catchy 等との違いは?

A. それぞれ位置が異なります。

- **Adobe Firefly / Canva** — デザインツールにAIが付いた形。当社は逆に「制作フロー全体を1本のパイプラインに組んだ」もの。
- **ChatGPT / Claude 単体** — 何でもできるが、ユーザーがプロンプトを書く必要がある。当社は「日本語でラフ指示するだけ」。
- **ELYZA / Catchy** — テキスト生成特化。当社は4アセット(コピー+画像+スクリプト+音声)を一気通貫。
- **Stable Diffusion + 手作業** — 自由度が高いが運用属人化。当社は標準化されたワークフロー。

**当社の独自性は「ディレクション翻訳」レイヤー**。ユーザー入力を構造化命令に変換する中間層がコアIPです。

#### Q5. なぜ日本市場で勝てると考えるのか?

A. 3点：

1. **日本語のディレクションを「設計の中心」に据えた数少ない製品**。海外発製品は英語前提で、日本語微妙ニュアンス(攻めた・落ち着いた・お祭り感・凛とした)の翻訳が浅い。
2. **国内SNS広告フォーマット(LINE、Yahoo!、Ameba)への出力対応**を優先実装可能。海外製品は対応が後回し。
3. **データ主権要件への適合**。日本の上場企業・金融機関・官公庁が求める「鍵を外部サーバーに渡さない」「監査証跡が取れる」を最初から満たす設計。

#### Q6. 競合の参入障壁は?半年で追いつかれないか?

A. UIだけ見れば模倣は可能です。ただし**ディレクション翻訳エンジンの精度**は、各アセット種別ごとに数十〜数百の事例で作り込まれた few-shot プロンプトの蓄積で決まります。

「攻めた感じ」というラフ指示を、「短文化、命令形、コントラスト強調、リムライト追加、暖色排除...」のような構造化命令にきちんと展開できるかどうか。ここは公開モデルが取って代わるものではなく、現場の制作監督と並走して磨き込む領域です。

加えて、**国内広告フォーマット対応・法令チェック(景表法・薬機法)・ブランド辞書**を積み上げることで、参入後発が追いつくのに 12〜18ヶ月の遅れが生じる構造にしていきます。

#### Q8. AIが間違ったコピーを生成して、炎上・苦情に繋がったら誰の責任?

A. **最終的な承認はすべて人間が行う設計**です。各ステップで「これでよいか」をユーザーが意思決定し、ディレクターズノートに記録されます。AIは「2案を出す」「批評する」「ラフ指示を構造化する」までで、決定権は持ちません。

エンタープライズ版では、加えて：

- **NGワード辞書による事前フィルタ**(景品表示法上の優良誤認・有利誤認表現等)
- **承認フロー**(役職別)
- **生成内容の完全な監査ログ**(誰が・いつ・何を承認したか)

を標準提供します。
